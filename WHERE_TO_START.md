## ğŸ”§ CORE COMPONENTS AT A GLANCE

### 1. CrossChannelSynthesis (NEW - Main Orchestrator)
**File:** `brd_agent/cross_channel_synthesis.py`
**What:** Coordinates the entire pipeline
**Use:**
```python
from brd_agent import CrossChannelSynthesis
synthesis = CrossChannelSynthesis()
brd = synthesis.synthesize_from_files()
```

### 2. BRDExtractionEngine (LLM-Powered)
**File:** `brd_agent/backend.py`
**What:** Extracts structured requirements
**Use:**
```python
from brd_agent import quick_extract
result = quick_extract("We need the API ready by March...")
```

### 3. DataIngestionEngine (Multi-Source Loader)
**File:** `brd_agent/data_ingest.py`
**What:** Loads from Enron, AMI, CSV, APIs
**Use:**
```python
from brd_agent import DataIngestionEngine
engine = DataIngestionEngine()
emails = engine.load_enron()
```

### 4. Configuration
**File:** `brd_agent/config.py`
**What:** Centralized settings and API keys
**Edit:** `.env` file to customize

### 5. Web/API Interfaces
**Files:** `brd_agent/api.py`, `brd_agent/frontend.py`
**Use:**
```bash
# REST API
python -m brd_agent.api

# Streamlit UI
streamlit run brd_agent/frontend.py
```

## â“ COMMON QUESTIONS

### Q: Do I need API keys?
**A:** No! Demo works without them. LLM extraction features need API keys for better results.

### Q: What if I don't have the Enron dataset?
**A:** Demo automatically uses sample data. Works great for testing!

### Q: How long does the full pipeline take?
**A:** Demo with samples: ~30 seconds. Real Enron data: ~5-10 minutes depending on data size.

### Q: Can I use this with my own data?
**A:** Yes! Upload CSV/JSON files or integrate your API in `data_ingest.py`

### Q: What's the difference between conflicts in emails vs meetings?
**A:** See `BRD_AGENT_IMPLEMENTATION_GUIDE.md` - "CRITICAL CONFLICT" section

### Q: How do I customize noise filtering?
**A:** Edit `NOISE_KEYWORDS` and `RELEVANCE_KEYWORDS` in `brd_agent/config.py`
